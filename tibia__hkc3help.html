Introduction:
Tired of waiting for CIP to implement a solution to the hotkey problem?
Tired of losing your entire configuration when watching/playing an older cam/OT?
Tired of not being able to use the same set of mapfiles when using different Config Paths?
Or, tired of having to set your VIP icons again for every single config if going by copypasting?
HKC3 can solve all those problems - and more.

HKC3 is a handy little application that allows you to use up to 10 different sets of hotkeys at the same time. You can switch from one to another as simply as double-clicking a desktop shortcut. Note, however, that you cannot change hotkeys during actual gameplay but have to close Tibia first. But! you can switch to another set (for an MC client) while you already have others running, with the newly implemented MC mode. VIP icons etc? Everything auto-saved to all the other sets of course.


Setup:
Close Tibia, extract the .rar, put the HKC3 folder including all of its content into your Tibia folder (eg: C:\Program Files\Tibia\HKC3\) and run hkc3.bat once.

Setup will create 9 additional hotkey sets as copies of your current configuration.
A backup of your mapfiles will be made and you'll find 3 new shortcuts on your desktop: One to the main menu of HKC3 and two more to switch to Set 1 and Set 2 respectively. You can create more or customise those two at any time later on.


Quickstart:
All right, installation finished. Your active hotkey set will be number 1. Let's switch to set 2.
Close all running Tibia clients and run the "Hotkey Set 2" shortcut on your desktop.
Start Tibia and log in. Your hotkeys will be the same still (since all new sets are copies of the intial config).
For illustration, change one of the hotkeys, eg. make F1 say "THIS IS SET 2".
Close Tibia again, run the "Hotkey Set 1" shortcut. When switching, the previous set is saved automatically.
Open up Tibia and you'll see your test hotkey is not changed here but in set 2 only.
And that's how HKC3 works.

VIP data like icons and comments will be made global, meaning it will be the same for all of your sets.
Just remember one thing: Before switching sets you MUST close all running Tibia clients first.
Unless, you use MC Mode.


MC mode:
The most important addition in version 3.
With MC mode you can switch sets whenever you want, no need to close Tibia before switching sets anymore.
You can, for example, have Tibia running with set 4, then switch to set 6 and open a new client with that.
On the downside, MC mode will NOT autosave upon switching sets. To save your configs with MC mode enabled, either open the HKC3 main menu and select "Save current config" OR use a SAVE desktop shortcut. For more information on the latter look at the next chapter of this manual.
Note that when saving, the saved config will be that of last closed client.

You can enable or disable MC mode from the HKC3 main menu by going to the "Advanced Options" menu.
After disabling MC mode no data will be saved upon your first switching sets.


Shortcuts and Titles:
To create a new Shortcut open up the HKC3 main menu and select "Create Desktop Shortcut".
Choose the hotkey set you want the shortcut to activate and an icon.
The new shortcut will by default be labelled "Hotkey Set ".
You can label your hotkey sets by giving them "Titles" in the "Advanced Options" menu.
Not only does this help you find your way through switch/save menus, but it also affects default shortcut names.

Then there's the SAVE shortcut:
It saves your current config (that of the last closed Tibia client) to one of the sets. By default it will save to your last opened set, but you'll be able to choose a different destination set. The save menu will display the first couple of hotkeys of the set you are about to save to, so you can make sure you're saving to the right set.


Silent/Boot mode:
When Silent mode is enabled the DOS box closes automatically after switching sets or saving your config instead of waiting for you to press a key. This is particularly handy when combined with Boot mode, which will automatically start up Tibia after switching to another hotkey set (which is why you have to put the HKC folder into your Tibia folder, else there's no way for me to find the right Tibia on your computer really).


Automatic Config Restore:
Another new, and really needed, feature. Upon switching sets HKC3 will check the size of your current Tibia.cfg file. If it is smaller than 1024byte (1kb) switching sets will be aborted, no data will be saved and the previous config restored. No data loss. This is to avoid major fuckups when accidentally resetting your config by playing/watching an <OLDER VERSION> OT/cam while having a <LASTEST VERSION> hotkey set enabled.

Getting stuck in an infinite loop trying to set up an <OLDER VERSION> set?
Pretty much impossible with the size used for this check. But if you still manage to do so, just change 2-3-4 "useless" hotkeys like Shift+F7 or the fuck do I know to "AAAAAAAAAAAAAAAAAAAAAAAAAAAAAA" or something like that, to push up the filesize, then close the client and save the set manually just to make sure.


Backup/Restore Function
Pretty much self-explanatory. Only difference to the HKC2 version of it is that you can make backups of your mapfiles as well. And that it doesn't save in a random folder on C:\ anymore but inside ApplicationData like the rest of your HKC3 data.


QNAASICJSFAQ (Questions never actually asked so I can't just say FAQ)
Q: Why still Batch?
A: It's the pretty much fastest and easiest way to handle something like this. Besides it offers transparency, meaning you can look at the source to be sure it's safe.

Q: Why do I have to put the HKC folder into my Tibia folder?
A: To make sure Boot mode works properly and you don't get a "Tibia.exe" not found error. Pretty sure not even half of you will use it, but I'm also sure if I hadn't done it like that, I could spam "PUT IT INTO TIBIA FOLDER IF YOU WANT IT TO WORK" all over this manual and people still wouldn't get it. People are idiots, so you have to force them not to fuck up. This won't be necessary anymore with HKC4.

Q: I switched to another set while Tibia was running (MC mode off), what can I do?
A: Simple answer: Switch back right away and nothing will happen. Less simple answer: The config file is read into the client whenever you open a new instance of Tibia and not saved back before you close that (or another) client again. You do the maths.

Q: What is XXMKLINK.EXE?
A: XXMKLINK is a tool by Pixelab Inc. (www.xxcopy.com) that allows the creation of shortcuts under batch, which under normal circumstances is not possible.

Q: So I went to Advanced Options -> Show AppData folders... what's that?
A: If you have to ask, it's probably best if you just stay away from there.

Q: What about compatibility with future updates?
A: HKC3 is generally forward compatible, unless CIP makes major changes to the .cfg files.
